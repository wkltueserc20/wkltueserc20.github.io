## Why

為了簡化「育兒助手」的功能，移除使用頻率較低的「尿便紀錄」功能，並將視覺重心轉移至寶寶的「營養（奶量）」與「發育（成長）」數據。這能提升介面的簡潔度與使用者體驗。

## What Changes

- **移除**：從系統中完全移除 `'diaper'` (尿布) 紀錄類型及其相關的所有 UI 組件（包含表單切換、狀態選擇、統計卡片）。
- **視覺調整 (方案 B)**：
    - 將首頁頂部的統計卡片改為兩列佈局。
    - 第一列（全寬）：顯示當日總奶量。
    - 第二列（兩欄）：左側顯示當日總睡眠時間，右側顯示最新的成長數據（體重/身高）。
- **數據相容性**：
    - **BREAKING**: 移除 `'diaper'` 紀錄類型。
    - 系統在讀取舊的 `LocalStorage` 資料或匯入 CSV 時，將自動忽略類型為 `diaper` 的紀錄。

## Capabilities

### New Capabilities
- `growth-summary`: 在首頁頂部摘要區塊顯示最新的成長數據（體重與身高）。

### Modified Capabilities
- `record-tracking`: 移除尿布紀錄功能，僅保留餵奶、睡眠與成長。
- `data-visualization`: 移除尿布相關的統計與圖表過濾。
- `data-sync`: 在匯入與同步邏輯中過濾掉尿布類型數據。

## Impact

- `src/App.tsx`: 大量的 UI 條件渲染、狀態管理與統計邏輯需要修改。
- `RecordType`: 類型定義變更。
- CSV 匯入邏輯：需要新增類型過濾防護。
